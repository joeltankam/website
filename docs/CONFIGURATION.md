# Site Configuration

This document explains how the centralized site configuration works in this project.

## Overview

All site metadata, social links, and branding information are centralized in `site.config.ts` at the project root. This makes it easy to:

- Update site information in one place
- Override values for different environments (staging, production)
- Maintain consistency across all pages
- Enable easy customization for other users

## Site Configuration File

**Location:** `site.config.ts`

```typescript
export const siteConfig = {
  hostname: process.env.VITE_HOSTNAME || 'https://joeltankam.com',
  title: "Joel Tankam's Blog",
  description: 'Articles on software engineering, technology, and innovation.',
  author: 'Joel Tankam',
  language: 'en-us',
  social: {
    twitter: '@joeltankam',
    github: 'joeltankam',
    linkedin: 'joeltankam'
  }
}

export function getHostname(mode?: string): string {
  if (mode === 'production' || process.env.NODE_ENV === 'production') {
    return siteConfig.hostname
  }
  return window?.location?.origin || siteConfig.hostname
}
```

## Environment Variables

You can override the hostname using the `VITE_HOSTNAME` environment variable:

```bash
# .env.production
VITE_HOSTNAME=https://your-custom-domain.com
```

This is useful for:
- Deploying to different domains (staging vs production)
- Testing with different base URLs
- Multi-tenant deployments

## Files Using Site Config

The following files have been updated to use `siteConfig`:

### Build Configuration
- **vite.config.js** - Passes hostname to sitemap and RSS plugins
- **vite-plugin-rss.ts** - Uses title, description, language for feed generation

### Views
- **src/views/Home.vue** - Uses author name and social links
- **src/views/About.vue** - Uses author name and social links
- **src/views/Blog.vue** - Uses author name and description
- **src/views/BlogPost.vue** - Uses author name and social links in structured data

### What's Configured

1. **Hostname** - Base URL for sitemap, RSS feeds, canonical URLs
2. **Title** - Site name used in feeds and meta tags
3. **Description** - Site tagline used in feeds and some SEO meta tags
4. **Author** - Your name, used throughout the site
5. **Language** - Content language (used in RSS/Atom feeds)
6. **Social Links** - GitHub, Twitter/X, LinkedIn usernames

## Customization

To customize the site for your own use:

1. Edit `site.config.ts`
2. Update the values to match your information
3. Rebuild the project: `npm run build`

Example:

```typescript
export const siteConfig = {
  hostname: process.env.VITE_HOSTNAME || 'https://yourdomain.com',
  title: "Your Name's Blog",
  description: 'Your tagline or site description.',
  author: 'Your Name',
  language: 'en-us',
  social: {
    twitter: '@yourhandle',
    github: 'yourusername',
    linkedin: 'yourusername'
  }
}
```

## Benefits

✅ **Single Source of Truth** - Update once, apply everywhere  
✅ **Type Safety** - TypeScript ensures correct usage  
✅ **Environment Flexibility** - Easy to deploy to different domains  
✅ **Consistency** - Impossible to have mismatched values across pages  
✅ **Maintainability** - Easy to update author info or social links  
✅ **Reusability** - Makes the project easy to fork and customize  

## Static Files

Note: `index.html` contains some static meta tags that are used for the initial page load. These are overridden by the dynamic SEO meta tags generated by the `useSeo` composable in each component, which do use `siteConfig` values.

For production builds, ensure all environment variables are properly set and the build process completes successfully.
